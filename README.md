# PulseCRM — сравнение трёх фронтенд-реализаций

Учебный проект (диплом): одна общая **REST + WebSocket** инфраструктура на **FastAPI** и три независимых SPA-клиента (**React**, **Vue 3**, **Svelte 5**), сознательно отличающихся **архитектурой интерфейса** и используемыми **сильными сторонами** каждого фреймворка.

Предметная область — упрощённая **мини-CRM** в духе Jira / Bitrix24: проекты, роли участников, расширенные статусы задач (канбан), типы задач, спринты, учёт времени (work log), лента активности, уведомления и глобальный поиск.

## Стек

| Слой | Технологии |
|------|------------|
| API | Python 3.12+, FastAPI, SQLAlchemy 2 async, PostgreSQL (asyncpg), JWT |
| React | Vite, React 18, React Router, **Recharts**, **@dnd-kit** |
| Vue | Vite, Vue 3, Vue Router, **Ant Design Vue** |
| Svelte | Vite, Svelte 5, svelte-spa-router, кастомная вёрстка |

## Запуск

### Переменные

Бэкенд читает `DATABASE_URL` (async PostgreSQL, например `postgresql+asyncpg://user:pass@localhost:5432/dbname`).  
Для локального Postgres без SSL часто нужно не включать SSL в клиенте — в коде уже есть отключение SSL по умолчанию (см. `backend/app/db/config.py`).

Фронтенды: `VITE_API_URL=http://localhost:4000` (или порт вашего uvicorn).

### Docker

```bash
docker compose up --build
```

По умолчанию бэкенд слушает порт **4000** (см. `docker-compose.yml`).

### Миграция существующей БД

Если база уже создана старой версией схемы, выполните:

```bash
cd backend
set DATABASE_URL=postgresql+asyncpg://...
python migrations/crm_v2.py
```

На «чистой» базе достаточно старта приложения: таблицы создаются через `create_all`.

## Функциональность API (кратко)

- **Аутентификация**: регистрация, логин JWT, профиль (`PATCH /auth/me`, поле `display_name`).
- **Проекты**: владелец, участники с ролями `ADMIN | MEMBER | VIEWER`, ключ префикса `key_prefix`.
- **Спринты**: вложены в проект (`/projects/{id}/sprints`).
- **Задачи**: статусы `BACKLOG | TODO | IN_PROGRESS | REVIEW | DONE`, типы `TASK | BUG | FEATURE | EPIC`, эпик/родитель (`parent_id`), спринт, story points, исполнитель, наблюдатели, категории.
- **Видимость**: задачи видят автор/владелец/исполнитель/наблюдатель и **все участники проекта** (включая наблюдение ленты комментариев).
- **Права редактирования**: автор или владелец задачи либо владелец проекта / участник с ролью `ADMIN` могут менять поля; исполнитель без этих прав может менять в основном **статус**.
- **Комментарии**, **чаты** проекта и глобальный чат + WebSocket.
- **Work log**: `GET/POST /tasks/{id}/work-logs`.
- **Активность**: `GET /activity/` (фильтры `task_id`, `project_id`).
- **Уведомления**: назначение исполнителя и др.
- **Поиск**: `GET /search?q=`.

## Три разных UX — зачем так сделано

### React (`frontend-react`)

**Идея:** интерфейс «продуктового» доска + аналитика.

- **Kanban** (`#/board`) на **@dnd-kit**: типичный паттерн для React — композиция небольших компонентов + сторонняя headless-библиотека для DnD без привязки к UI-kit.
- **Дашборд** на **Recharts**: декларативные графики в JSX, один источник данных → состояние React.
- **Компоновка**: фиксированный **левый сайдбар** + верхняя строка поиска и колокольчик уведомлений.

**Сильные стороны React**, которые здесь задействованы:

- Огромная экосистема готовых библиотек (DnD, графики).
- Предсказуемая модель «дерево компонентов + состояние» для сложных интерактивных экранов.
- Легко смешивать уровни абстракции (layout / страница / виджет).

**Типичные компромиссы:** больший runtime и объём бандла по сравнению со Svelte; без дисциплины легко получить лишние перерисовки (здесь объём умеренный).

---

### Vue (`frontend-vue`)

**Идея:** «коробочная» enterprise-CRM на базе **Ant Design Vue**.

- **Вертикальный layout**: `a-layout-sider` + `a-layout-header` — паттерн админок Bitrix24 / корпоративных порталов.
- **Глобальный поиск** и **уведомления** в шапке через компоненты Ant (`Input.Search`, `Popover`, `Badge`).
- Экран проекта расширен **типами задач** и полным набором статусов в форме создания.

**Сильные стороны Vue**, которые здесь задействованы:

- Однофайловые компоненты и **Composition API** — хорошая читаемость бизнес-логики экрана.
- Глубокая интеграция с **Ant Design Vue**: таблицы, формы, модалки, layout без ручной вёрстки дизайн-системы.
- Плавная кривая обучения и зрелость для корпоративных интерфейсов.

**Типичные компромиссы:** размер бандла из-за UI-kit; кастомный «не-ant» дизайн иногда сложнее, чем на чистом CSS.

---

### Svelte (`frontend-svelte`)

**Идея:** лёгкий «newsfeed» активности с упором на **анимации и минимальный runtime**.

- Главная страница — **лента событий** (`activity.feed`) с переходами `fly`.
- Навигация — **контрастный минималистичный top bar** с «таблетками», отличный от React-сайдбара и Vue-shell Ant.
- Почти без тяжёлых зависимостей (кроме роутера).

**Сильные стороны Svelte**, которые здесь задействованы:

- Компиляция в обычный JS → **меньше абстракций в рантайме**.
- Встроенные `transition:` / `animate:` — декларативные анимации без отдельной библиотеки.
- Очень компактный код экранов для типовых CRUD/лент.

**Типичные компромиссы:** экосистема меньше React; для сложного DnB-канбана чаще тянут сторонние решения или пишут больше низкоуровневого кода.

## Сводное сравнение (для главы диплома)

| Критерий | React | Vue | Svelte |
|----------|-------|-----|--------|
| Обучаемость | Высокая, много материалов | Высокая, мягкий вход | Высокая, другой синтаксис шаблонов |
| Экосистема | Максимальная | Большая | Средняя, растущая |
| Размер бандла | Средний / выше | Средний (особенно с Ant) | Обычно ниже |
| Производительность UI | Отлично при грамотном состоянии | Отлично | Очень хорошо «из коробки» |
| Корпоративные UI-kit | MUI, Ant (React), Chakra… | **Ant Design Vue**, Naive UI, Vuetify… | Skeleton, меньше «готовых гигантов» |
| Подходит для | Сложный интерактив, интеграции | Админки, формы, таблицы | Лёгкие SPA, анимации, перформанс |

**Вывод для проекта:** три клиента демонстрируют не «кто лучше вообще», а **какой инструмент выгоднее под конкретный UX-профиль** — доска с DnD и графиками (React), формально-табличная CRM (Vue + Ant), лента и минимализм (Svelte).
